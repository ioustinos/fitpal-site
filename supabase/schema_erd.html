Fitpal Ordering Platform — Database Schema (Final)

27 tables · All money in cents · Bilingual _el/_en · Reconciled 2026-04-15

User & prefs Menu & catalogue Orders & delivery Wallet & payments System & admin

Changes from design review & codebase audit (Apr 15)

Structural changes

  • profiles: removed goal_*, pref_*, pref_slot_mon, pref_addr_mon — split into dedicated tables
  • user_goals: new table — enabled toggle + min/max range per macro
  • user_prefs: new table — payment, cutlery, invoice, dietary, lang, newsletter, only_admin_orders
  • user_day_prefs: new table — per-day default address + time window (Mon–Fri)
  • tags: new table — free-form labels with bg_color + font_color for UI pills
  • dish_tags: new junction table — replaces array tags on dishes
  • zone_time_slots: new table — maps delivery zones to available time windows
  • meal_services: new table — employee-curated meal selection subscriptions

Column changes

  • child_orders: time_slot text → time_from (time) + time_to (time); dropped driver, stop_number, eta (stays in Airtable for now); dropped day_subtotal (derived from order_items)
  • profiles: added name_en, avatar_url; kept dietician, dietary_notes
  • addresses: added lat, lng for zone validation; added sort_order; kept is_default + area
  • dishes: dropped array tags (→ dish_tags), dropped enum temp (→ tags)
  • order_items: added variant_label_en, comment
  • orders: dropped voucher_code, voucher_discount (voucher_uses is now source of truth); kept discount_amount as total
  • vouchers: added nullable user_id for customer-linked vouchers; added remaining (int) for credit vouchers that deplete over multiple uses
  • voucher_uses: added amount (int) — how much discount was applied from this voucher on this order
  • user_prefs: added only_admin_orders boolean
  • wallet_plans: reworked from fixed tiers to purchase history — stores calculator snapshot (consumer_type, meals, people, days, frequency, cost, credits, bonus_pct, bonus_amount, bonus_expires_at)
  • wallets: now references active_plan_id; dropped discount_pct (lives on plan); added bonus expiry support
  • All tables: added created_at / updated_at timestamps

Auth strategy

  • Launch: Email + password with one-time email verification link
  • Later: Magic link (passwordless email), Phone + OTP (SMS via Twilio)
  • All auth managed by Supabase Auth — no custom auth tables needed
  • On signup trigger creates profiles, user_goals, user_prefs rows

Considered & rejected

  • Translations table instead of _el/_en columns — rejected (2 languages only, JOINs add latency, no practical benefit)
