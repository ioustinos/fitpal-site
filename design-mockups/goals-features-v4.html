Feature 1

Cart — goals module (both states)

Show per-day intake always. Hide the bars when goal tracking is off.
Module appears after cart items, per active day. Goals ON: 4 rows with coloured bars and "current / goal min–max" numbers. Goals OFF: same numbers as compact pills, no bars, no status colouring — purely informational.

Feature 2

Checkout — goals module mirrors the cart

Same component, same two states, rendered inside the checkout OrderSummary sidebar.
No separate component. Factor out `<GoalsModule />` in `src/components/shared/` and render it in both `CartSidebar` and `OrderSummary`. Reads cart state + `user.prefs.goalTracking` + `user.goals` — identical behaviour everywhere.

Feature 4

Account · Orders — per-day intake

Expanding an order reveals each child order with its own intake pills.
Since every Order is a parent of Child Orders (one per delivery date), the most natural place is an inline expansion. Pills use the same 4-macro icon+number pattern as the cart "goals off" state, regardless of whether goal tracking is on.

Feature 5

Goals page — Days · Weeks · Months · Forecast

Table-driven. Date filter + pagination (50/page; hide pager if under 50 rows).
Period switch changes the row grain: Days = 1 row per delivery date, Weeks/Months = averages across dates in that period, Forecast = upcoming delivery dates with planned-intake (from unfulfilled orders) + projected averages. All four use the same table shell.

Feature 6

Orders list — filter + pagination

Same filter+pager shell as goals; page size = 50, hidden when total < 50.
Date range + status chips + search bound on order number / dish name. URL sync for ?from=&to=&status=&page= so a user can share/bookmark a filtered view. Pager identical to goals.
